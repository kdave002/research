# Related Work Draft (Stage 2)

Research on context and prompt compression for large language models has progressed quickly, but the field is still fragmented across objectives. One line of work focuses on aggressive prompt shortening for black-box models, where the core question is how much compression can be achieved before downstream quality drops. A central reference is Girish et al. (2024), which frames prompt compression through a rate–distortion lens and computes distortion–rate limits for query-aware and query-agnostic compressors. That paper is extremely relevant because it establishes a principled compression frontier; however, its distortion objectives are output-level fidelity and task error proxies, not a typed decomposition of which input information elements are causally load-bearing for multi-hop reasoning. Our RCID program explicitly targets that missing layer.

A second neighboring line studies compression of generated reasoning traces rather than source memory context. Lee et al. (2025) show a robust length–accuracy tradeoff in chain-of-thought compression and introduce a token-complexity perspective that estimates minimum response length for successful solving. This is conceptually close to our threshold framing, but the intervention point differs: Lee et al. compresses output reasoning trajectories, while RCID compresses and ablates input memory chunks before reasoning. In other words, token complexity is about minimum generation budget; RCID is about minimum retained information structure in context.

Recent works on reasoning-path compression (e.g., Song et al., 2025) further strengthen this distinction. Reasoning-path compression optimizes inference throughput by shortening intermediate trajectories, often with minimal loss in final-answer accuracy. RCID is orthogonal: it asks what types of source facts and connectors (entities, temporal markers, causal links, numeric facts) must survive compression so that reasoning trajectories remain valid in the first place. This shift from trajectory efficiency to context criticality is the key novelty axis.

Agent memory papers such as MemR3 (Du et al., 2025) motivate the practical need for RCID. MemR3 emphasizes retrieval quality and reflective memory access in long-horizon agent settings, showing that memory retrieval policy strongly affects performance. Yet retrieval quality is still usually evaluated by relevance or final-task outcomes without a fine-grained accounting of which compressed memory components are indispensable for multi-hop consistency. RCID can complement retrieval frameworks by providing a typed criticality measure that explains why some retrieved memory works and some fails despite similar retrieval scores.

Broader context-pruning and chunking literature (query-aware chunking, adaptive windows, token pruning, hierarchical retrieval) demonstrates consistent gains in efficiency, but often relies on coarse heuristics and aggregate accuracy. The missing bridge is a structural metric that predicts collapse points and differentiates benign compression from reasoning-breaking compression. Our primary research question directly addresses this by mapping compression ratio to reasoning accuracy curve shape (linear, step, threshold) across genres. The secondary question identifies which information classes are disproportionately load-bearing relative to token share.

In summary, the literature supports three conclusions: (1) compression-accuracy tradeoffs are real and measurable, (2) both input-side and output-side compression can reduce cost, and (3) existing evaluation does not adequately capture typed reasoning-critical information loss. RCID is positioned as a unifying metric to fill this gap. By combining compression sweeps with information-type ablations and multi-hop evaluation, our work aims to provide an experimentally grounded, causally interpretable account of how much context can be forgotten before reasoning fails.
